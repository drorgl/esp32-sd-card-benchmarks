# SD Card Benchmarks
This is a thorough test of more and less familiar ESP32 SD Card access methods. 
My journey began when I discovered the same problem described in this (issue)[https://github.com/espressif/esp-idf/issues/3249].

This repository is related to my article (ESP32 SD Card Optimization)[https://blog.drorgluska.com/2022/06/esp32-sd-card-optimization.html]

# Tests:
* Write Sequential, variable block sizes
* Read Sequential, variable block sizes
* Write Random, variable block sizes
* Read Random, variable block sizes
* Write Random aligned, variable block sizes
* Read Random aligned, variable block sizes

# Data
* Write Time
* Read Time
* Block Size: 512 bytes, 1k, 2k, 4k, 8k, 16k, 32k, 64k

## Modified fatfs component
add exfat support in ffconf.h (under components/fastfs/src)
```
#define FF_FS_EXFAT		1
```
modify vfs_fat.c (under components/fatfs/vfs) to add support for (fast seek)[http://elm-chan.org/fsw/ff/doc/config.html#use_fastseek]
from:
```
if(!(fat_mode_conv(flags) & (FA_WRITE))) {
        DWORD *clmt_mem =  ff_memalloc(sizeof(DWORD) * CONFIG_FATFS_FAST_SEEK_BUFFER_SIZE);
```
to:
```
if((fat_mode_conv(flags) & (FA_READ))) {
    DWORD *clmt_mem =  ff_memalloc(sizeof(DWORD) * CONFIG_FATFS_FAST_SEEK_BUFFER_SIZE);
```

the meaning of this modification is whenever a file is opened for read (not just read as the original code states) it will attempt to build the fast seek map.

Note: this modification makes the behavior of esp32 non-standard, you may have to modify the flags used in your fopen.
